## 4) Data Handling Summary
- Authentication/profile data for access control.
- Operational record data entered by authorized users.
- Local temporary storage for offline continuity.
- Sync metadata (`device_id`, timestamps, retry/error status).

## 5) Permissions Explanation
- `INTERNET`: API auth/sync/report operations.
- File provider access: save/share generated reports (PDF).

## 6) Backup / Sync Explanation
- Pending writes are queued locally when offline.
- App pushes pending records when connectivity returns.
- Pull sync fetches server updates to keep mobile aligned with web.

## 7) Offline + Online Behavior
- Offline: user can continue entries; records are marked pending.
- Online restore: auto-sync runs and updates statuses.
- Failed/conflict records are shown to user with retry option.
